Android - 让控件变得可拖拽

发表于 2014-03-25 | 分类于 Android Notes

其实要实现拖拽的控件是非常简单的事情，和让控件支持点击一样简单！我们只需要实现onDragEvent() dispatchDragEvent() setOnDragListener()即可，是不是和触摸事件的回调差不多呢？(onTouchEvent() dispatchTouchEvent() setOnTouchListener())

我们可以通过View.startDarg(ClipData data, DragShadowBuilder shadowBuilder, Object myLocalState, int flags)来开始拖拽。
这里需要我们传入一个DragShadowBuilder对象，DragShadow就是拖拽阴影，会在你拖动的时候显示出来，
所以DragShadowBuilder有一个传View的构造函数-new DragShadowBuilder(View)。

Android - 使用Loader进行数据加载

发表于 2014-03-18 | 分类于 Android Notes

Android3.0 开始提出Loader和LoaderManager的概念，通过LoaderManager类可以很轻松的从Fragment或Activity 中进行异步加载数据。
这和AsyncTask很类似，但是AsyncTask没有LoaderManager的统一管理，并且不方便实现MVC的开发模式，对于和UI相关的数据加载就不要使用AsyncTask使用Loader最佳！
官方示例：
http://developer.android.com/reference/android/app/LoaderManager.html
http://developer.android.com/reference/android/content/AsyncTaskLoader.html
当然我们实际开发中，需要自己去实现一个更加酷的Loader，来满足我们的开发需要。

总结这段时间的面试经验

发表于 2014-03-08 | 分类于 刘说八道

2014.02.17-2014.02.28 这两周一直在面试，找工作，前前后后面试了十多家公司。
当然绝大多数都是成都的小型创业公司，一般创业公司多为直接面试，好一点的公司或者对技术比较看中的公司会有笔试。
大牛看到勿喷，本文仅仅作为我的一点点经验，应该能代表百分之五十的软件专业毕业生吧。

一、准备工作

简历

1.网投平台一般有简历模板，这个要尽量填满，填完之后预览，看哪里要修改的，修改需要突出显示的。(比如说我就把我的博客信息，直接填在专业描述里面，因为这样可以显示在最前面)
2.邮件投简历，需要你自己做一个简历文档，找模板，注意最后最好导出成PDF格式，不要用Doc。
3.通知面试最好备一份纸质简历，有些公司可能不会把你发给他的简历打印出来。

笔试

1.临时抱佛脚吧，直接上网找笔试题，然后做，一般找个两三套就够了。
2.Java主要知识点:线程同步，异常处理，集合(Collection的那些子类，Arraylist和LinkedList的不同)，多态性，final, finally, finalize的不同。

面试

1.有笔试，可能会对你笔试内容做个询问。
2.会让你介绍你做过的项目。
3.基本上所有公司都有问Android Listview的优化，Bitmap的处理，网络异步通讯。
